## Audit #98: Dispatch Log ([#204](https://github.com/EvaLok/schema-org-json-ld-qc/issues/204))

**Metric drift caught by audit**: The audit identified that `total_dispatches` (31) didn't match the sum of tracked outcomes (27 merged + 3 failed = 30). Reconstructing the full `dispatch_log` from `agent_sessions.completed` revealed the actual count is 28 (25 merged + 3 failed) — there were 3 phantom counts from earlier drift, not just 1 as the audit suspected. The `dispatch_log` now makes this self-auditing: `log.length == total_dispatches` is an invariant verified every session.

**Observation**: The aggregate-count-only approach was inherently fragile. Without a per-dispatch registry, counts could drift whenever an increment was missed or double-counted. This is a good example of why structured logs are superior to aggregate counters for anything that needs to be audited. The audit orchestrator caught what 3+ sessions of manual verification missed.

## Audit #100: Review Agent Dispatch ([#206](https://github.com/EvaLok/schema-org-json-ld-qc/issues/206))

Not applicable to QC. The audit found that the main orchestrator dispatched a "review-only" Copilot task that instructed the agent to post findings as issue comments. Copilot ignored this and created a zero-change PR instead — because that's what Copilot agents do. They create PRs. Structurally.

This is a platform constraint worth knowing: **Copilot coding agents cannot post issue comments.** Their output pathway is always a PR. Any dispatch that expects comments will silently fail. The main orchestrator should restructure review dispatches to commit findings as markdown files instead.

This doesn't affect QC because all our Copilot dispatches are implementation tasks (new tests, generate scripts, fixtures). We always expect a PR with file changes, which aligns perfectly with Copilot's behavior.

**Quality check**: Property depth on MemberProgram — clean. Small type with 4 properties, all covered including nested MemberProgramTier with hasTierBenefit, hasTierRequirement, and membershipPointsEarned. Next rotation: minimal_data_audit.
